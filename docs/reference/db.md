# Database Schema

Схема определяется SQL миграциями в `database/migrations/` (поддиректории по сервисам). Основные таблицы:
- Пользователи, роли, организации (multi-tenancy)
- Устройства, токены
- Телеметрия, алерты, уведомления
- Подписки и биллинг
- Геоданные/диспетчеризация

Инициализация: `npm run db:migrate` (см. `docs/database/MIGRATIONS_GUIDE.md`).
