# ⚠️ ВАЖНО: Изменение портов приложений

## Измененные порты:

- **Guardian App**: `5173` → `5174` ✅
- **Dispatcher App**: `5174` → `5175` ✅

## Что нужно сделать:

### 1. Перезапустить приложения:

```bash
# Остановите все запущенные приложения (Ctrl+C)
# Затем запустите заново:

npm run dev:guardian   # Теперь на порту 5174
npm run dev:dispatcher # Теперь на порту 5175
```

### 2. Обновить ссылки в браузере:

- **Guardian App**: http://localhost:5174 (было 5173)
- **Dispatcher App**: http://localhost:5175 (было 5174)

### 3. Закрыть старые вкладки:

Если у вас открыты старые вкладки с портами 5173 или 5174, закройте их и откройте новые с правильными портами.

### 4. Очистить кеш браузера (если нужно):

Если проблемы продолжаются:
- Нажмите `Ctrl+Shift+R` (Windows/Linux) или `Cmd+Shift+R` (Mac) для жесткой перезагрузки
- Или очистите кеш браузера

## Если видите ошибку "ERR_CONNECTION_REFUSED" на порту 5173:

Это означает, что:
1. Приложение не запущено на старом порту 5173
2. Нужно открыть новую вкладку с портом 5174
3. Или перезапустить guardian-app

## Проверка:

После перезапуска проверьте:
- Guardian App: http://localhost:5174
- Dispatcher App: http://localhost:5175
